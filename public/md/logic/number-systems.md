# 진수 설명

진수는 수를 어떤 기준으로 표현할지 정하는 방법입니다. 우리가 평소에 쓰는 수는 10진수이고, 컴퓨터 내부에서는 2진수가 기본입니다. 디지털 논리회로에서는 전압의 높고 낮음을 `0`과 `1`로 구분하기 때문에 2진수를 잘 이해하는 것이 중요합니다.

이 문서에서는 연산보다 **표현과 변환**에 집중합니다. 덧셈, 뺄셈, 보수 같은 계산은 다음 문서인 [2진수 연산](/logic/binary-arithmetic)에서 따로 다룹니다.

목표는 다음과 같습니다.

- 진수가 무엇인지 설명할 수 있다.
- 자릿값 원리를 이해한다.
- 2진수, 8진수, 10진수, 16진수를 구분할 수 있다.
- 2진수와 10진수를 서로 변환할 수 있다.
- 2진수와 8진수, 2진수와 16진수를 빠르게 변환할 수 있다.
- 소수 부분을 2진수로 표현하는 기본 원리를 이해한다.

한 문장으로 정리하면 이렇습니다.

> 진수는 자릿값의 기준이고, 어떤 진수를 쓰느냐에 따라 한 자리에서 사용할 수 있는 숫자와 자리의 무게가 달라진다.

---

## 1. 진수란?

진수는 한 자리에서 사용할 수 있는 숫자의 개수입니다.

10진수는 한 자리에 `0`부터 `9`까지 10개의 숫자를 사용합니다. 2진수는 한 자리에 `0`과 `1`만 사용합니다.

| 진수 | 사용하는 숫자 | 예시 |
|------|---------------|------|
| 2진수 | `0`, `1` | `1011(2)` |
| 8진수 | `0`~`7` | `27(8)` |
| 10진수 | `0`~`9` | `25(10)` |
| 16진수 | `0`~`9`, `A`~`F` | `1A(16)` |

진수를 표시할 때는 보통 수 뒤에 괄호로 진수를 붙입니다.

```text
1011(2)  -> 2진수 1011
27(8)    -> 8진수 27
25(10)   -> 10진수 25
1A(16)   -> 16진수 1A
```

프로그래밍에서는 다음 표기법도 자주 사용합니다.

| 표현 | 의미 |
|------|------|
| `0b1011` | 2진수 1011 |
| `0o27` | 8진수 27 |
| `25` | 10진수 25 |
| `0x1A` | 16진수 1A |

## 2. 왜 컴퓨터는 2진수를 쓸까?

컴퓨터는 내부에서 전기 신호를 사용합니다. 전압을 아주 세밀하게 여러 단계로 나누어 구분하면 오류가 생기기 쉽습니다. 그래서 컴퓨터는 보통 두 상태만 구분합니다.

- 전압이 낮은 상태: `0`
- 전압이 높은 상태: `1`

이렇게 두 상태만 사용하면 회로가 단순해지고, 잡음이 조금 섞여도 값을 안정적으로 구분할 수 있습니다.

예를 들어 어떤 신호가 정확히 `3.000V`여야만 1이라고 정하면 작은 잡음에도 오류가 생깁니다. 하지만 `2.5V`보다 높으면 1, 낮으면 0처럼 구간으로 판단하면 훨씬 안정적입니다.

그래서 컴퓨터는 문서, 사진, 음악, 숫자, 명령어까지 결국 모두 0과 1의 긴 나열로 표현합니다.

## 3. 자릿값

진수를 이해할 때 가장 중요한 것은 자릿값입니다. 10진수 `253`을 보겠습니다.

```text
253(10) = 2*100 + 5*10 + 3*1
        = 2*10^2 + 5*10^1 + 3*10^0
```

오른쪽 끝 자리는 `10^0`, 그 왼쪽은 `10^1`, 그 왼쪽은 `10^2`입니다.

2진수도 원리는 같습니다. 기준이 10이 아니라 2일 뿐입니다.

```text
1011(2) = 1*2^3 + 0*2^2 + 1*2^1 + 1*2^0
        = 1*8 + 0*4 + 1*2 + 1*1
        = 11(10)
```

16진수도 같은 방식입니다.

```text
1A(16) = 1*16^1 + A*16^0
       = 1*16 + 10*1
       = 26(10)
```

핵심은 이것입니다.

```text
오른쪽 끝 자리 = 진수^0
왼쪽으로 한 칸 이동 = 진수를 한 번 더 곱함
```

## 4. 16진수의 A부터 F

16진수는 한 자리에서 16개의 값을 표현해야 합니다. 그런데 숫자 모양은 `0`부터 `9`까지 10개뿐입니다. 그래서 10부터 15까지는 알파벳을 사용합니다.

| 16진수 | 10진수 값 |
|--------|-----------|
| A | 10 |
| B | 11 |
| C | 12 |
| D | 13 |
| E | 14 |
| F | 15 |

예를 들어 `F(16)`은 10진수 `15`입니다. `10(16)`은 10진수 `10`이 아니라 16진수의 두 자리 수이므로 10진수 `16`입니다.

```text
F(16)  = 15(10)
10(16) = 16(10)
1F(16) = 1*16 + 15 = 31(10)
```

여기서 많이 헷갈리는 지점은 `10`이라는 모양입니다. `10`은 어느 진수로 읽느냐에 따라 값이 달라집니다.

| 표기 | 10진수 값 |
|------|-----------|
| `10(2)` | 2 |
| `10(8)` | 8 |
| `10(10)` | 10 |
| `10(16)` | 16 |

## 5. 2진수에서 10진수로 변환

2진수를 10진수로 바꿀 때는 각 자리의 값을 더하면 됩니다.

예를 들어 `110101(2)`를 10진수로 바꿔봅시다.

| 자리 | 2^5 | 2^4 | 2^3 | 2^2 | 2^1 | 2^0 |
|------|-----|-----|-----|-----|-----|-----|
| 값 | 32 | 16 | 8 | 4 | 2 | 1 |
| 비트 | 1 | 1 | 0 | 1 | 0 | 1 |

`1`이 있는 자리만 더합니다.

```text
110101(2) = 32 + 16 + 4 + 1
          = 53(10)
```

또 다른 예시입니다.

```text
10000000(2) = 128(10)
11111111(2) = 128 + 64 + 32 + 16 + 8 + 4 + 2 + 1
            = 255(10)
```

8비트에서 모두 `1`이면 unsigned 기준으로 `255`가 됩니다.

## 6. 10진수에서 2진수로 변환

10진수를 2진수로 바꿀 때는 2로 계속 나누고, 나머지를 아래에서 위로 읽습니다.

예를 들어 `45(10)`를 2진수로 바꿔봅시다.

| 나누기 | 몫 | 나머지 |
|--------|----|--------|
| 45 / 2 | 22 | 1 |
| 22 / 2 | 11 | 0 |
| 11 / 2 | 5 | 1 |
| 5 / 2 | 2 | 1 |
| 2 / 2 | 1 | 0 |
| 1 / 2 | 0 | 1 |

나머지를 아래에서 위로 읽으면 `101101`입니다.

```text
45(10) = 101101(2)
```

검산해보면 다음과 같습니다.

```text
101101(2) = 32 + 8 + 4 + 1
          = 45(10)
```

이 방법은 모든 양의 정수에 사용할 수 있습니다. 몫이 0이 될 때까지 계속 2로 나누는 것이 핵심입니다.

## 7. 2진수와 16진수 변환

16진수는 2진수를 짧게 쓰기 위해 자주 사용합니다. 이유는 16이 `2^4`이기 때문입니다.

즉, 2진수 4비트는 16진수 1자리와 정확히 대응됩니다.

| 2진수 | 16진수 |
|-------|--------|
| 0000 | 0 |
| 0001 | 1 |
| 0010 | 2 |
| 0011 | 3 |
| 0100 | 4 |
| 0101 | 5 |
| 0110 | 6 |
| 0111 | 7 |
| 1000 | 8 |
| 1001 | 9 |
| 1010 | A |
| 1011 | B |
| 1100 | C |
| 1101 | D |
| 1110 | E |
| 1111 | F |

예를 들어 `10110110(2)`를 16진수로 바꿔봅시다.

```text
10110110(2)
= 1011 0110
= B    6
= B6(16)
```

반대로 `3F(16)`을 2진수로 바꾸면 각 자리를 4비트로 풀면 됩니다.

```text
3F(16)
= 3    F
= 0011 1111
= 00111111(2)
```

앞쪽의 `0`은 값에는 영향을 주지 않지만, 8비트처럼 자리 수를 맞춰야 할 때는 중요합니다.

```text
3F(16) = 111111(2)
8비트로 쓰면 00111111(2)
```

## 8. 2진수와 8진수 변환

8진수는 2진수 3비트와 정확히 대응됩니다. 이유는 8이 `2^3`이기 때문입니다.

| 2진수 | 8진수 |
|-------|-------|
| 000 | 0 |
| 001 | 1 |
| 010 | 2 |
| 011 | 3 |
| 100 | 4 |
| 101 | 5 |
| 110 | 6 |
| 111 | 7 |

예를 들어 `110101(2)`를 8진수로 바꿔봅시다.

```text
110101(2)
= 110 101
= 6   5
= 65(8)
```

만약 비트 수가 3개씩 나누어떨어지지 않으면 왼쪽에 `0`을 붙여 맞춥니다.

```text
10110(2)
= 010 110
= 2   6
= 26(8)
```

## 9. 자주 쓰는 2의 거듭제곱

2진수와 다른 진수를 빠르게 바꾸려면 2의 거듭제곱을 자주 봐두는 것이 좋습니다.

| n | 2^n |
|---|-----|
| 0 | 1 |
| 1 | 2 |
| 2 | 4 |
| 3 | 8 |
| 4 | 16 |
| 5 | 32 |
| 6 | 64 |
| 7 | 128 |
| 8 | 256 |
| 9 | 512 |
| 10 | 1024 |
| 16 | 65536 |

특히 `2^8 = 256`은 중요합니다. 8비트로 표현할 수 있는 서로 다른 값의 개수는 256개입니다.

| 비트 수 | 표현 가능한 패턴 개수 |
|---------|-----------------------|
| 1비트 | 2개 |
| 2비트 | 4개 |
| 4비트 | 16개 |
| 8비트 | 256개 |
| 16비트 | 65536개 |

## 10. 소수 부분의 2진수 표현

정수뿐 아니라 소수도 2진수로 표현할 수 있습니다. 소수점 오른쪽은 `2^-1`, `2^-2`, `2^-3`처럼 작아집니다.

| 자리 | 값 |
|------|----|
| 2^-1 | 1/2 = 0.5 |
| 2^-2 | 1/4 = 0.25 |
| 2^-3 | 1/8 = 0.125 |
| 2^-4 | 1/16 = 0.0625 |

예를 들어 `0.101(2)`는 다음과 같습니다.

```text
0.101(2) = 1*0.5 + 0*0.25 + 1*0.125
         = 0.625(10)
```

`10.101(2)`라면 정수 부분과 소수 부분을 따로 계산합니다.

```text
10.101(2) = 2 + 0.625
          = 2.625(10)
```

소수를 10진수에서 2진수로 바꿀 때는 2를 계속 곱하고, 정수 부분을 순서대로 읽습니다.

예를 들어 `0.625(10)`를 바꿔봅시다.

| 계산 | 정수 부분 | 남은 소수 |
|------|-----------|-----------|
| 0.625 * 2 = 1.25 | 1 | 0.25 |
| 0.25 * 2 = 0.5 | 0 | 0.5 |
| 0.5 * 2 = 1.0 | 1 | 0 |

따라서 다음과 같습니다.

```text
0.625(10) = 0.101(2)
```

주의할 점은 모든 10진수 소수가 2진수로 딱 떨어지지는 않는다는 것입니다. 예를 들어 `0.1(10)`은 2진수로 무한히 반복됩니다.

## 11. 변환 문제 푸는 순서

진수 변환 문제는 다음 순서로 풀면 실수가 줄어듭니다.

1. 현재 수가 몇 진수인지 확인한다.
2. 바꿀 목표 진수가 무엇인지 확인한다.
3. 2진수와 16진수는 4비트씩 묶는다.
4. 2진수와 8진수는 3비트씩 묶는다.
5. 10진수로 바꿀 때는 자릿값을 더한다.
6. 10진수에서 다른 진수로 바꿀 때는 목표 진수로 나누고 나머지를 읽는다.

예를 들어 `10111100(2)`를 16진수로 바꾸는 문제라면 4비트씩 묶는 것이 가장 빠릅니다.

```text
10111100(2)
= 1011 1100
= B    C
= BC(16)
```

## 12. 자주 하는 실수

진수 문제에서 자주 나오는 실수는 다음과 같습니다.

- 16진수 `A`를 11로 착각한다. `A`는 10이다.
- 16진수 `10`을 10진수 10으로 읽는다. `10(16)`은 16이다.
- 2진수 4비트를 16진수 1자리로 묶을 때 오른쪽부터 묶지 않는다.
- 8진수 변환에서 3비트씩 묶어야 하는데 4비트씩 묶는다.
- 10진수에서 2진수로 바꿀 때 나머지를 위에서 아래로 읽는다.
- 앞쪽의 0은 값에는 영향이 없지만, 비트 수를 맞출 때는 필요하다는 점을 놓친다.

특히 묶어서 변환할 때는 항상 오른쪽부터 묶어야 합니다.

```text
잘못된 묶기: 10 1101 10
올바른 묶기: 1011 0110
```

## 13. 핵심 요약

- 진수는 자릿값의 기준이다.
- 오른쪽 끝 자리는 항상 `진수^0`이다.
- 2진수는 `0`, `1`만 사용한다.
- 8진수는 `0`부터 `7`까지 사용한다.
- 16진수는 `0`부터 `9`, `A`부터 `F`까지 사용한다.
- 16진수 한 자리는 2진수 4비트와 대응된다.
- 8진수 한 자리는 2진수 3비트와 대응된다.
- 10진수에서 2진수로 바꿀 때는 2로 나누고 나머지를 아래에서 위로 읽는다.
- 소수점 오른쪽의 2진수 자리는 `1/2`, `1/4`, `1/8`처럼 작아진다.
