# 공통 기초 - 포인터

포인터는 **변수의 주소를 저장하는 변수**이다. 시스템 프로그래밍, C 언어 모두에서 핵심 개념이며 메모리와 자원을 직접 다룬다.

---

## 메모리 구조 이해

일반적인 프로세스의 가상 주소 공간은 다음과 같이 나뉜다.

```
낮은 주소
[코드 영역]
[전역/정적 영역]
[힙 영역]      <- malloc/free
...
[스택 영역]    <- 함수 호출, 지역 변수
높은 주소
```

* **힙**: 동적 할당 메모리
* **스택**: 함수 호출 시 생기는 지역 변수

---

## 포인터 기본

```c
int x = 10;
int *p = &x;   // x의 주소를 저장

printf("x=%d\n", x);   // 10
printf("*p=%d\n", *p); // 10
```

* `&` : 주소 연산자
* `*` : 역참조(주소가 가리키는 값 접근)

---

## 포인터와 배열

```c
int arr[3] = {1, 2, 3};
int *p = arr; // arr == &arr[0]

printf("%d\n", *(p + 1)); // 2
```

* `p + 1`은 다음 요소의 주소
* 배열은 포인터처럼 동작한다

---

## 동적 메모리와 포인터

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {
    int *p = (int *)malloc(sizeof(int) * 3);
    if (!p) return 1;

    p[0] = 10; p[1] = 20; p[2] = 30;

    printf("%d %d %d\n", p[0], p[1], p[2]);

    free(p);
    return 0;
}
```

* `malloc`은 힙에 메모리를 할당
* 반드시 `free`로 해제해야 메모리 누수를 막는다

---

## 시스템 프로그래밍에서 자주 쓰는 포인터 패턴

### 1. void 포인터

```c
void *vp = NULL; // 어떤 타입 주소든 저장 가능
```

* 범용 포인터
* 사용할 때는 반드시 타입 캐스팅 필요

### 2. 구조체 포인터

```c
typedef struct {
    int fd;
    int flags;
} File;

File f = {3, 1};
File *pf = &f;

printf("fd=%d\n", pf->fd); // 구조체 포인터 접근
```

* `->` 연산자는 구조체 포인터 멤버 접근

---

## 자주 발생하는 버그

* **널 포인터 역참조**: `NULL`을 가리키는 포인터 사용
* **댕글링 포인터**: `free` 후 포인터를 계속 사용
* **초기화되지 않은 포인터**: 쓰레기 주소 접근
* **범위 초과 접근**: 배열 범위를 넘어 접근

---

## 핵심 정리

* 포인터는 **주소를 저장하는 변수**
* 메모리와 자원을 다루기 위해 필수
* 동적 메모리 사용 시 반드시 `malloc/free`를 짝으로 관리
