# 2. 프로세스 관찰

프로세스는 실행 중인 프로그램입니다. 파일로 저장된 `a.out`은 아직 프로그램 파일일 뿐이고, 실제로 실행되어 메모리에 올라가면 프로세스가 됩니다.

### 2.1 `ps`

```bash
ps
```

현재 터미널과 관련된 프로세스를 보여 줍니다. 보통 다음과 비슷한 열이 보입니다.

```text
PID TTY          TIME CMD
1234 pts/0    00:00:00 bash
1301 pts/0    00:00:00 ps
```

중요한 열은 다음과 같습니다.

- `PID`: 프로세스 번호
- `TTY`: 연결된 터미널
- `TIME`: CPU를 사용한 누적 시간
- `CMD`: 실행된 명령

`ps`를 실행할 때마다 `ps` 자체도 새 프로세스로 만들어집니다. 그래서 매번 `ps`의 PID가 달라지는 것이 자연스럽습니다.

### 2.2 `ps -ef`

```bash
ps -ef
```

시스템 전체 프로세스를 더 자세히 보여 줍니다.

자주 보는 열은 다음과 같습니다.

- `UID`: 실행한 사용자
- `PID`: 현재 프로세스 번호
- `PPID`: 부모 프로세스 번호
- `C`: CPU 사용량 관련 값
- `STIME`: 시작 시간
- `TTY`: 터미널
- `TIME`: CPU 사용 시간
- `CMD`: 실행 명령

반복적으로 나타나는 정보는 보통 `UID`, `PID`, `PPID`, `CMD`입니다. 프로세스는 항상 누군가가 실행했고, 부모 프로세스와 연결되어 있기 때문입니다.

### 2.3 `top`

```bash
top
```

실시간으로 프로세스 상태를 보여 줍니다. 종료는 `q`입니다.

`top`에서 자주 보는 항목은 다음과 같습니다.

- `%CPU`: CPU 사용률
- `%MEM`: 메모리 사용률
- `PID`: 프로세스 번호
- `USER`: 실행 사용자
- `COMMAND`: 실행 명령

CPU 사용률이 높은 프로세스는 그 순간 CPU를 많이 쓰는 작업입니다. 시간이 지나면 값이 바뀌는 이유는 운영체제가 프로세스를 번갈아 스케줄링하고, 각 프로그램이 하는 일이 계속 변하기 때문입니다.

### 2.4 `pstree`

```bash
pstree
pstree -p
```

프로세스의 부모-자식 관계를 나무 모양으로 보여 줍니다.

`pstree -p`를 쓰면 PID도 함께 보입니다.

```text
systemd(1)-+-sshd(700)---sshd(1220)---bash(1221)---pstree(1302)
```

이 구조에서 `bash`가 `pstree`를 실행했기 때문에 `bash`가 부모, `pstree`가 자식입니다.
