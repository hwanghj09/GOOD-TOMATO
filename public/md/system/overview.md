# 전체 지도

이 문서는 `image` 디렉터리의 실습 프린트를 바탕으로, 수업에서 나온 흐름을 한 번에 복습할 수 있게 정리한 자료입니다.

목표는 세 가지입니다.

- 리눅스에서 프로세스를 관찰하고 제어할 수 있다.
- C 언어로 포인터, 동적 메모리, 프로세스 생성, 실행 교체, 종료를 이해한다.
- 시그널과 파이프를 이용해 프로세스끼리 통신하는 기본 구조를 설명하고 구현한다.

읽는 순서는 위에서 아래로 가면 됩니다. 문서 중간중간에는 사이트 광고가 자연스럽게 들어가도록 표시를 넣어 두었습니다.

---

시스템 프로그래밍은 운영체제 가까이에서 동작하는 프로그램을 만드는 공부입니다. 일반적인 C 문법만 쓰는 것이 아니라, 운영체제가 제공하는 기능을 직접 호출합니다.

이때 자주 만나는 핵심 단어는 다음과 같습니다.

- **프로세스**: 실행 중인 프로그램
- **PID**: 프로세스 식별 번호
- **PPID**: 부모 프로세스의 PID
- **권한**: 누가 파일을 읽고, 쓰고, 실행할 수 있는지 정하는 규칙
- **포인터**: 메모리 주소를 저장하는 변수
- **힙**: `malloc`으로 실행 중에 빌려 쓰는 메모리 영역
- **스택**: 함수 호출과 지역 변수가 저장되는 메모리 영역
- **fork**: 현재 프로세스를 복제해서 자식 프로세스를 만드는 함수
- **exec**: 현재 프로세스의 프로그램 내용을 다른 프로그램으로 바꾸는 함수
- **wait**: 부모가 자식의 종료를 기다리는 함수
- **signal**: 프로세스에게 보내는 짧은 이벤트/명령
- **pipe**: 한 프로세스의 출력이 다른 프로세스의 입력으로 흘러가게 하는 통로

한 문장으로 정리하면 이렇습니다.

> 시스템 프로그래밍은 프로세스, 메모리, 파일, 권한, 통신을 운영체제 API로 직접 다루는 공부다.
