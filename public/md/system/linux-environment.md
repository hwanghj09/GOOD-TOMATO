# 1. 리눅스 접속과 기본 환경 확인

프린트의 첫 실습은 서버에 접속해서 "내가 지금 어느 시스템에서 누구로 작업 중인가"를 확인하는 흐름입니다.

### 1.1 SSH 접속

```bash
ssh 사용자이름@서버IP
```

예를 들어 사용자 이름이 `student`이고 서버 IP가 `192.168.0.10`이면 다음처럼 접속합니다.

```bash
ssh student@192.168.0.10
```

접속하면 내 컴퓨터의 터미널처럼 보이지만, 실제 명령은 서버에서 실행됩니다.

확인할 점은 다음과 같습니다.

- 프롬프트가 바뀌었는가?
- `hostname` 결과가 내 컴퓨터 이름이 아니라 서버 이름으로 나오는가?
- `pwd`의 현재 위치가 서버의 홈 디렉터리인가?

### 1.2 사용자와 위치 확인

```bash
whoami
hostname
pwd
```

각 명령의 뜻은 다음과 같습니다.

- `whoami`: 현재 로그인한 사용자 이름
- `hostname`: 현재 접속한 컴퓨터/서버 이름
- `pwd`: 현재 작업 중인 디렉터리의 절대 경로

처음에는 이 세 명령을 습관처럼 치는 것이 좋습니다. 서버 실습에서 가장 흔한 실수는 "내가 어디에 접속해 있는지"를 착각하는 것입니다.

### 1.3 기본 파일 명령

```bash
ls
ls -l
cd 디렉터리명
mkdir practice
touch test.txt
cat test.txt
rm test.txt
```

특히 `ls -l`은 권한, 소유자, 그룹, 파일 크기, 수정 시간을 한 번에 보여 주므로 시스템 프로그래밍에서 자주 사용합니다.

```text
-rw-r--r-- 1 user user 12 May 12 21:00 test.txt
```

이 중 앞의 `-rw-r--r--`가 권한 정보입니다.
