# 4. 사용자, 그룹, 권한

시스템 프로그래밍에서는 프로그램이 "무엇을 할 수 있는가"가 중요합니다. 같은 코드라도 어떤 사용자로 실행하느냐에 따라 파일을 읽을 수 있거나 없을 수 있습니다.

### 4.1 현재 사용자 확인

```bash
whoami
id
groups
```

- `whoami`: 사용자 이름
- `id`: UID, GID, 그룹 목록
- `groups`: 내가 속한 그룹 목록

리눅스는 사용자 이름만 보는 것이 아니라 내부적으로 숫자 ID를 사용합니다.

- `UID`: 사용자 ID
- `GID`: 기본 그룹 ID

### 4.2 권한 구조

`ls -l`을 실행하면 다음과 같은 형식이 보입니다.

```text
-rw-r--r-- 1 user class 20 May 12 21:00 test.txt
```

권한 부분을 나누면 다음과 같습니다.

```text
- rw- r-- r--
  소유자 그룹 기타사용자
```

각 문자의 의미는 다음과 같습니다.

- `r`: read, 읽기
- `w`: write, 쓰기
- `x`: execute, 실행
- `-`: 해당 권한 없음

파일 종류 문자는 맨 앞에 붙습니다.

- `-`: 일반 파일
- `d`: 디렉터리
- `l`: 심볼릭 링크

### 4.3 숫자 권한

권한은 숫자로도 표현합니다.

- 읽기 `r` = 4
- 쓰기 `w` = 2
- 실행 `x` = 1

그래서 `644`는 다음 뜻입니다.

```text
6 = 4 + 2 = rw-
4 = 4     = r--
4 = 4     = r--
```

즉 `644`는 `rw-r--r--`입니다.

자주 쓰는 권한은 다음과 같습니다.

- `600`: 소유자만 읽고 쓰기 가능
- `644`: 소유자는 읽기/쓰기, 나머지는 읽기
- `700`: 소유자만 읽기/쓰기/실행
- `755`: 소유자는 전부 가능, 나머지는 읽기/실행
- `777`: 모두 전부 가능, 실습 외에는 위험

### 4.4 `chmod`

```bash
chmod 000 test.txt
cat test.txt
chmod 644 test.txt
cat test.txt
```

`chmod 000`이면 아무도 읽고 쓰고 실행할 수 없습니다. 보통 `cat test.txt`에서 `Permission denied`가 납니다.

다시 `chmod 644`를 주면 소유자는 읽고 쓸 수 있고, 다른 사용자는 읽을 수 있습니다.

### 4.5 디렉터리 권한의 핵심

파일의 `x`는 실행 권한입니다. 디렉터리의 `x`는 "그 디렉터리 안으로 들어가거나 경로 탐색을 할 수 있는 권한"입니다.

그래서 디렉터리는 보통 `755` 또는 `700`을 많이 씁니다.

```bash
chmod 700 private_dir
chmod 755 public_dir
```

### 4.6 소유자 변경

```bash
chown user file.txt
chgrp group file.txt
sudo chown user:group file.txt
```

일반 사용자는 보통 마음대로 소유자를 바꿀 수 없습니다. 필요하면 `sudo`가 필요합니다.
