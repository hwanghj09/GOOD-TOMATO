# 24. 마지막 압축 정리

시스템 프로그래밍의 큰 흐름은 다음 한 줄입니다.

```text
리눅스에서 사용자/권한을 확인하고, C로 메모리를 다루며, fork/exec/wait로 프로세스를 만들고, signal/pipe로 제어와 통신을 한다.
```

정말 최소로 외울 것은 다음입니다.

- `whoami`, `hostname`, `pwd`: 내가 누구이고 어디인지 확인
- `ps`, `top`, `pstree`: 프로세스 관찰
- `chmod`, `ls -l`: 권한 확인과 변경
- `malloc`, `free`: 힙 메모리 관리
- `fork`: 프로세스 복제
- `exec`: 프로그램 교체
- `wait`: 자식 종료 회수
- `kill`, `signal`: 시그널 전송과 처리
- `pipe`, `read`, `write`, `dup2`: 프로세스 간 통신

프린트 문제를 풀 때는 항상 이렇게 생각하면 됩니다.

1. 이 명령/코드가 새 프로세스를 만드는가?
2. PID와 PPID는 어떻게 바뀌는가?
3. 부모와 자식이 같은 코드를 모두 실행하는가?
4. 메모리는 공유되는가, 복사되는가?
5. 권한 때문에 실패할 가능성이 있는가?
6. 읽기/쓰기 fd를 제대로 닫았는가?
7. 종료 상태를 부모가 회수했는가?

이 질문에 답할 수 있으면 시스템 프로그래밍의 뼈대는 잡힌 것입니다.
