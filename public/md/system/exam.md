# 20. 시험/수행평가용 핵심 답안

### 20.1 프로세스란?

프로세스는 실행 중인 프로그램이다. 운영체제는 각 프로세스에 PID, 메모리 공간, 열린 파일, 실행 상태 등을 부여하고 CPU를 스케줄링한다.

### 20.2 PID와 PPID란?

PID는 프로세스를 구분하는 번호이고, PPID는 그 프로세스를 만든 부모 프로세스의 PID이다.

### 20.3 `ps`와 `top`의 차이

`ps`는 특정 시점의 프로세스 목록을 보여 주고, `top`은 프로세스 상태를 실시간으로 갱신해서 보여 준다.

### 20.4 `fork`란?

현재 프로세스를 복제하여 자식 프로세스를 만드는 시스템 호출이다. 부모와 자식은 `fork` 이후의 코드를 각각 실행한다.

### 20.5 `exec`란?

현재 프로세스의 프로그램 이미지를 다른 프로그램으로 교체하는 시스템 호출이다. 성공하면 기존 코드로 돌아오지 않는다.

### 20.6 `fork`와 `exec`의 차이

`fork`는 프로세스를 새로 복제하고, `exec`는 현재 프로세스를 새로운 프로그램으로 바꾼다.

### 20.7 `wait`가 필요한 이유

부모가 자식 프로세스의 종료를 기다리고 종료 상태를 회수하기 위해 필요하다. `wait`하지 않으면 좀비 프로세스가 남을 수 있다.

### 20.8 시그널이란?

시그널은 프로세스에게 전달되는 이벤트나 제어 메시지이다. 예를 들어 `Ctrl + C`는 `SIGINT`, `kill PID`는 기본적으로 `SIGTERM`을 보낸다.

### 20.9 `SIGTERM`과 `SIGKILL`의 차이

`SIGTERM`은 정상 종료 요청이라서 프로그램이 처리하거나 무시할 수 있다. `SIGKILL`은 처리하거나 무시할 수 없으며 운영체제가 강제로 종료한다.

### 20.10 파이프란?

파이프는 한 프로세스가 쓴 데이터를 다른 프로세스가 읽을 수 있게 해 주는 단방향 통신 통로이다. `fd[1]`에 쓰고 `fd[0]`에서 읽는다.

### 20.11 포인터란?

포인터는 메모리 주소를 저장하는 변수이다. `&`는 주소를 얻고, `*`는 주소가 가리키는 값을 접근한다.

### 20.12 `malloc`과 `free`

`malloc`은 힙에 동적 메모리를 할당하고, `free`는 할당한 메모리를 반납한다. `free`하지 않으면 메모리 누수가 생길 수 있다.

### 20.13 권한 `644`의 의미

`644`는 소유자는 읽기/쓰기 가능, 그룹과 기타 사용자는 읽기만 가능하다는 뜻이다. 문자로는 `rw-r--r--`이다.
