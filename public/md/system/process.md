# 7. 프로세스 개념

프로그램과 프로세스는 다릅니다.

- **프로그램**: 디스크에 저장된 실행 파일
- **프로세스**: 실행되어 메모리에 올라간 프로그램의 인스턴스

같은 프로그램을 여러 번 실행하면 같은 실행 파일에서 여러 프로세스가 만들어질 수 있습니다.

### 7.1 프로세스가 가진 것

프로세스는 보통 다음 정보를 가집니다.

- PID
- PPID
- 실행 중인 코드
- 독립된 가상 주소 공간
- 열린 파일 목록
- 현재 작업 디렉터리
- 사용자/그룹 권한
- 레지스터와 프로그램 카운터 같은 CPU 실행 상태

운영체제는 이 정보를 바탕으로 여러 프로세스를 동시에 실행되는 것처럼 관리합니다.

### 7.2 프로세스 상태

프로세스는 계속 CPU를 잡고 있는 것이 아닙니다.

- Running: 실행 중
- Ready: 실행 가능하지만 CPU 대기
- Waiting/Blocked: 입력, 파일, 네트워크 등 이벤트 대기
- Stopped: 시그널 등으로 정지
- Zombie: 종료했지만 부모가 아직 종료 상태를 회수하지 않음

`top`, `ps`의 상태 열에서 이런 상태를 볼 수 있습니다.

### 7.3 PID와 PPID 확인 코드

```c
#include <stdio.h>
#include <unistd.h>

int main(void) {
    printf("PID  = %d\n", getpid());
    printf("PPID = %d\n", getppid());
    return 0;
}
```

컴파일과 실행은 다음처럼 합니다.

```bash
gcc pid.c -o pid
./pid
./pid
./pid
```

실행할 때마다 PID는 보통 달라집니다. 매번 새로운 프로세스가 만들어지기 때문입니다.

PPID는 보통 현재 셸의 PID입니다. 같은 터미널에서 계속 실행하면 PPID는 같게 나올 수 있습니다.
