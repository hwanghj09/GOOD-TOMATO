# 3. 백그라운드 실행과 프로세스 종료

프린트에는 `sleep 100 &`, `ps`, `kill [PID]` 흐름이 나옵니다. 이 실습은 "프로세스가 살아 있는 상태를 확인하고, 원하는 프로세스만 종료하는 법"을 익히는 것입니다.

### 3.1 백그라운드 실행

```bash
sleep 100 &
```

`sleep 100`은 100초 동안 아무것도 하지 않고 기다리는 명령입니다. 뒤에 `&`를 붙이면 터미널을 붙잡지 않고 백그라운드에서 실행됩니다.

실행하면 보통 다음처럼 보입니다.

```text
[1] 23456
```

여기서 `23456`은 백그라운드로 실행된 `sleep` 프로세스의 PID입니다.

### 3.2 PID 찾기

```bash
ps
ps -ef | grep sleep
```

`ps` 결과에서 `sleep` 프로세스의 PID를 찾습니다.

```text
23456 pts/0 00:00:00 sleep
```

### 3.3 프로세스 종료

```bash
kill 23456
```

`kill`은 이름이 조금 무섭지만, 기본 의미는 "프로세스에게 시그널을 보낸다"입니다. 기본 시그널은 `SIGTERM`이고, 이것은 "정상 종료해 달라"는 요청입니다.

다시 `ps`를 실행했을 때 `sleep`이 사라졌다면 종료된 것입니다.

### 3.4 강제 종료

```bash
kill -9 23456
```

`-9`는 `SIGKILL`입니다. 이 시그널은 프로세스가 처리하거나 무시할 수 없습니다. 운영체제가 바로 종료시킵니다.

하지만 평소에는 먼저 `kill PID`를 쓰고, 종료되지 않을 때만 `kill -9 PID`를 쓰는 습관이 좋습니다. `SIGKILL`은 프로그램이 파일 저장, 임시 파일 정리, 자원 해제 같은 마무리 작업을 할 기회를 주지 않습니다.

### 3.5 작업 제어

```bash
jobs
fg %1
bg %1
Ctrl + Z
Ctrl + C
```

- `jobs`: 현재 셸에서 돌고 있는 백그라운드/정지 작업 확인
- `fg %1`: 1번 작업을 포그라운드로 가져오기
- `bg %1`: 1번 작업을 백그라운드에서 계속 실행
- `Ctrl + Z`: 현재 포그라운드 작업 일시 정지
- `Ctrl + C`: 현재 포그라운드 작업에 `SIGINT` 보내기
