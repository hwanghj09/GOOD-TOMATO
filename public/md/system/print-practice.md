# 16. 프린트 실습 흐름별 정리

이미지 프린트에 나온 실습을 흐름대로 정리하면 다음과 같습니다.

### 16.1 서버 접속

실행:

```bash
ssh 사용자이름@서버IP
```

관찰:

- 접속 전후 프롬프트가 달라진다.
- 서버에서 명령이 실행된다.
- `hostname`으로 현재 시스템을 확인할 수 있다.

핵심:

> SSH는 내 컴퓨터에서 원격 서버의 셸을 사용하는 방법이다.

### 16.2 사용자 및 환경 확인

실행:

```bash
whoami
hostname
pwd
```

관찰:

- `whoami`는 현재 사용자
- `hostname`은 현재 접속 시스템
- `pwd`는 현재 디렉터리

핵심:

> 시스템 실습 전에는 항상 사용자, 시스템, 현재 위치를 확인한다.

### 16.3 프로세스 확인

실행:

```bash
ps
ps -ef
```

관찰:

- `ps`는 현재 터미널 중심
- `ps -ef`는 시스템 전체에 가까운 목록
- PID, PPID, USER, CMD가 중요하다.

핵심:

> 프로세스는 PID로 구분하고, PPID로 부모-자식 관계를 추적한다.

### 16.4 실시간 관찰

실행:

```bash
top
```

종료:

```text
q
```

관찰:

- CPU 사용률이 계속 변한다.
- 프로세스 상태가 실시간으로 바뀐다.

핵심:

> 운영체제는 여러 프로세스를 계속 스케줄링하므로 상태와 사용률은 시간에 따라 변한다.

### 16.5 프로세스 구조

실행:

```bash
pstree
pstree -p
```

관찰:

- 프로세스가 나무 구조로 연결되어 있다.
- 셸이 명령을 실행하면 셸이 부모, 명령이 자식이 된다.

핵심:

> 리눅스의 프로세스는 부모-자식 관계를 가진다.

### 16.6 프로세스 생성과 종료

실행:

```bash
sleep 100 &
ps
kill PID
ps
```

관찰:

- `&`를 붙이면 백그라운드에서 실행된다.
- `sleep` 프로세스의 PID를 찾을 수 있다.
- `kill` 후 목록에서 사라진다.

핵심:

> 프로세스는 PID로 찾아 제어할 수 있다.

### 16.7 예제 실행 결과 확인

실행:

```bash
gcc example.c -o example
./example
./example
./example
```

관찰:

- 실행할 때마다 PID는 달라진다.
- 같은 터미널에서 실행하면 PPID는 같을 수 있다.
- fork 예제는 출력 순서가 달라질 수 있다.

핵심:

> 매 실행은 새 프로세스이며, fork 이후의 실행 순서는 스케줄러에 따라 달라질 수 있다.

### 16.8 권한 실험

실행:

```bash
chmod 000 test.txt
cat test.txt
chmod 644 test.txt
cat test.txt
```

관찰:

- 권한이 없으면 `Permission denied`가 난다.
- 권한을 복구하면 다시 읽을 수 있다.

핵심:

> 파일 접근 가능 여부는 코드만이 아니라 사용자와 권한에 의해 결정된다.

### 16.9 `exec` 실험

코드:

```c
printf("변신 전: 현재 프로세스입니다.\n");
execlp("ls", "ls", "-l", NULL);
printf("이 문구가 보인다면 exec가 실패한 것입니다.\n");
```

관찰:

- `execlp` 성공 후에는 아래 `printf`가 실행되지 않는다.
- 현재 프로세스가 `ls` 프로그램으로 바뀐다.

핵심:

> `fork`는 복제, `exec`는 교체다.
